¿Qué es un JavaScript Framework?

En el ámbito del desarrollo del software, este término hace referencia a un conjunto de bibliotecas que proporcionan herramientas que ayudan a desarrollar y organizar sitios web o aplicaciones.

Se trata de un entorno o marco de trabajo que consta de un conjunto de prácticas, conceptos y criterios a seguir estandarizados.

Millones de desarrolladores usan frameworks para acelerar el flujo de trabajo de desarrollo y aplicar las mejores prácticas de una manera fluida y fácil.

Debido a su gran variedad, su uso depende de los objetivos principales, la funcionalidad general de la plataforma, los requisitos del proyecto y cómo se puede implementar dentro de cada escenario específico.

Al usar este tipo de marcos de desarrollo para JavaScript, se puede ahorrar una gran cantidad de tiempo y esfuerzo en el desarrollo de sitios web y aplicaciones basados en este lenguaje, simplificando todo el procedimiento y permitiendo a los desarrolladores crear aplicaciones web a gran escala de manera eficiente.

¿Por qué usar Javascript Frameworks?

Cuando en 1996 surge Javascript, le da algo de interactividad a una web que hasta el momento estaba compuesta por documentos estáticos.

Sin embargo, a medida que la web evolucionaba y la popularidad de Javascript crecía, las y los desarrolladores comenzaron a escribir herramientas para resolver problemas y las compartían con otros en forma de paquetes reutilizables llamados librerías.

Hoy en día, Javascript es una parte esencial de la web y la web es parte esencial de la vida cotidiana.

Esto no sería posible sin frameworks, ya que estos ayudan a construir webs más dinámicas e interactivas, de forma más simple, eficiente, fácil de mantener y escalar.

¿Frameworks o Bibliotecas?

Algunas herramientas consideradas Frameworks, como React, son realmente de bibliotecas, pero, ¿cuál es la diferencia?

Bibliotecas

  • Conjunto de funciones, clases y métodos codificados en un lenguaje de programación, que ofrece una interfaz o un contrato definido para una funcionalidad específica.
  • Su objetivo principal es ejecutar una acción concreta para reutilizar el código.
  • Permite llamar funciones específicas.
  • Generalmente permiten solucionar un problema concreto.
  • Con una biblioteca estás a cargo del código de tu aplicación, utilizandola solo para añadir nuevas funcionalidades.

Frameworks

  • Marco de trabajo que a veces trae un conjunto de librerías y “sub-frameworks” que consiguen que el desarrollo de una aplicación sea más rápido.
  • Definen el diseño de la aplicación, brindando un set de herramientas para desarrollar sistemas o aplicaciones.
  • Son los mismos frameworks los que llaman un bloque de código de una manera predeterminada.
  • No ofrece una sola solución sino que brinda una plantilla completa para desarrollar una aplicación.
  • Un framework te limita en el sentido de que es necesario trabajar en torno al marco y reglas que éste brinda.
