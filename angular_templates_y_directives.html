Templates

Cada componente tiene un template HTML que declara como un componente se renderiza. El desarrollador define este template mediante código o ruta de archivo.

Angular agrega elementos de sintaxis que extienden el HTML para que se puedan insertar valores dinámicos desde el propio componente.

Directives

Directives extienden la funcionalidad del HTML usando para ello una nueva sintaxis. Con ella podemos usar lógica que será ejecutada en el DOM (Document Object Model). Agregan nuevo comportamiento o modifican el existente de los elementos en el template. Cada Directiva que usamos tiene un nombre, y determina donde puede ser usada, sea en un elemento, atributo, componente o clase.

Se crean ejecutando el comando CLI ng generate directive nombre-directiva

Un Component es una directive con template. @Component es un decorator @Directive extendido con características propiedas de los templates.

Structural directives (ngIf, ngFor, ngSwitch)

  • ngIf esta directiva se utiliza cuando quieres mostrar algún bloque HTML solo si cumple con ciertas condiciones.
  • ngFor: se utiliza para cuando necesites que un bloque se repita varias veces.
  • ngSwitch: esta directiva te permite agregar una sentencia switch con varios valores y casos para seleccionar.

Cuando se aplican, generalmente adelante llevan un asterisco *, ejemplo *ngIf. Esta abreviación, Angular la interpreta y la convierte a una forma más larga, a un <ng-template> que rodea al elemento "huésped" y a sus descendientes.

Sólo se puede aplicar UNA structural directive a un elemento, en caso contrario el compilador lo tomaría como un error. La razón: simplicidad, structural directives realizan tareas complejas.

Attribute directives (NgClass, NgStyle, NgModel)

  • NgClass: esta directiva añade estilo condicional utilizando una clase.
  • NgStyle: añade estilo condicional en la línea.
  • NgModel: Esto te permite hacer un enlace bidireccional. Esto significa que puedes pasar datos en ambas direcciones entre el archivo HTML y TypeScript. Ejemplo, muestra una propiedad de datos, y se actualiza cuando el usuario realiza cambios.
