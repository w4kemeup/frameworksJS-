Dependency Injection

La inyección de dependencia es un patrón de diseño en el que una clase solicita dependencias de fuentes externas en lugar de crearlas, extrayendo la responsabilidad de creación de instancias de un componente para delegarla en otro.

Existen dos roles principales en este sistema: el que consume dependencias y el que provee dependencias. Angular facilita la comunicación entre estos roles a través de una abstracción llamada inyector.

Servicios y componentes

Angular distingue los componentes de los servicios para aumentar la modularización y la reutilización. Al separar la funcionalidad relacionada con la vista de un componente de otros tipos de procesamiento, puedes hacer que tus componentes sean ágiles y eficientes.

Un servicio es una categoría amplia que abarca cualquier valor, función o característica que necesite una aplicación. Un servicio es típicamente una clase con un propósito limitado y bien definido. Debe hacer algo específico y hacerlo bien.
Un componente se encarga de permitir la experiencia del usuario. Un componente debe presentar propiedades y métodos para el enlace de datos, para mediar entre la vista (representada por la plantilla) y la lógica de la aplicación (que a menudo incluye alguna noción de modelo).

Un componente puede delegar ciertas tareas a los servicios, como obtener datos del servidor, validar la entrada del usuario o registrar información directamente en la consola. Al definir tales tareas de procesamiento en una clase de servicio inyectable, hace que esas tareas sean disponibles para cualquier componente.

Los componentes consumen servicios, es posible inyectar un servicio en un componente, dándole al componente acceso a ese servicio.

Dependencias

Las dependencias son servicios u objetos que una clase necesita para cumplir su función.

Proveer dependencias

En este ejemplo, se está definiendo la clase HeroService para que actúe de dependencia de un componente. Esto se hace con el decorador @Injectable

Lo siguiente es hacer que esa clase recién creada quede disponible, para lo cual hay varias formas:

A nivel del componente, usando el campo providers del decorador @Component. En este caso el servicio creado queda disponible para todas las instancias de este componente y otros componentes y directivas utilizadas en esta plantilla.

A nivel de NgModule, también usando el campo providers, pero del decorador @NgModule. En este caso, el servicio está disponible para todos los componentes, directivas y pipes declarados en este NgModule.

A nivel root de la aplicación, lo cual se hace agregandole el campo providedIn: 'root' al decorador @Injectable

Inyectar dependencias

La forma más común de inyectar una dependencia es declararla en un constructor de clase. Cuando Angular crea una nueva instancia de un componente, directiva o pipe, determina qué servicios u otras dependencias requiere al fijarse en los tipos de parámetros del constructor.
